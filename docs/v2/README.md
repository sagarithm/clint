# ColdMailer V2 Documentation Hub

This folder contains the advanced implementation package for turning ColdMailer into a high-ticket client acquisition engine.

## Document Map
- 00-master-implementation-plan.md: Strategic blueprint and 90-day roadmap
- 01-source-connectors-and-filters.md: Multi-source ingestion and qualification logic
- 02-developer-todo.md: Sprint-based engineering checklist
- 03-operating-guidelines.md: Weekly operating system and execution policy
- 04-system-architecture.md: Service boundaries, queues, and runtime flow
- 05-database-schema-v2.md: Data model for source intelligence and lifecycle tracking
- 06-campaign-playbooks.md: Source-specific messaging and conversion playbooks
- 07-deliverability-and-email-safety.md: Inbox health and sender risk controls
- 08-ai-prompt-and-personalization-spec.md: Prompt architecture and quality guardrails
- 09-kpi-dashboard-and-experiments.md: KPI framework and A/B testing system
- 10-launch-checklists.md: Pre-launch, launch, and post-launch checklists

## How To Use
1. Align on ICP and offer first (00, 03).
2. Implement schema and architecture foundations (04, 05).
3. Build source connectors and scoring engine (01, 02).
4. Launch campaigns with strict quality and deliverability controls (06, 07, 08).
5. Optimize weekly with KPI and experiment loops (09, 10).

## Ownership Recommendation
- Founder/BDM: strategy, offer, proof assets
- Engineering: architecture, connectors, orchestration, reliability
- Growth Ops: campaign execution, quality control, suppression
- Marketing: messaging experiments and conversion optimization
